# Prompt 4 — Marking-menu для планет Cosmos

## Что изменено
- Для планет в `Cosmos` добавлено жестовое радиальное меню (marking-menu) по удержанию.
- Короткий tap/click оставлен как раньше: выбор планеты + обычное компактное меню (fallback).
- Конфликт переключателей подписей устранён через взаимоисключение:
  - при включении «Показывать все подписи» автоматически выключается «Показывать только важные»;
  - при включении «Показывать только важные» автоматически выключается «Показывать все подписи».

## Жесты и поведение

### Tap / click (короткий)
- Работает как раньше: открывает обычное меню действий около планеты.

### Hold (long press / click-and-hold)
- Если удержание длится `~210ms`, открывается радиальное меню вокруг точки удержания.
- Пока radial-menu активно, pan/zoom карты временно блокируется.

### Mark (жест выбора)
- После открытия радиального меню движение указателя выбирает сектор по углу.
- Параметры:
  - `holdDelay`: `210ms`
  - `deadzoneRadius`: `12px`
  - `activationRadius`: `22px`
- Если палец/курсор не вышел из deadzone к отпусканию — команда не выполняется (только показ меню).
- Если сектор выбран и указатель отпущен — выполняется соответствующая команда.

## Команды в radial/menu
Минимальный набор команд:
- **Открыть** — всегда доступно.
- **Данные** — только если для планеты есть `dataHref`.
- **Результат** — только если у острова есть route результата и уже есть `lastReport`.
- **Отмена** — закрывает меню без перехода.

## Mobile / Desktop
- **Mobile**: touch-hold открывает radial; быстрый tap оставляет обычное меню.
- **Desktop**: mouse click-and-hold открывает radial; обычный click оставляет обычное меню.
- Клавиатура: `Enter`/`Space` по выбранной планете открывают обычное меню (fallback).

## Доступность
- Для SVG-планет сохранён `role="button"` и `aria-label`.
- Для элементов действий обычного и радиального меню добавлены `aria-label`.
