# Prompt 13 — Ensemble cashflow forecast + model disagreement

## Что добавлено

Для прогноза `net cashflow` теперь доступны 2 режима:

- **Быстрый (1 метод)** — только IID bootstrap.
- **Надёжный (ансамбль)** — смесь из трёх простых методов.

Методы в ансамбле:

1. **IID bootstrap**: сэмплинг по месяцам из последних N наблюдений.
2. **Moving Block Bootstrap (MBB)**: сэмплинг блоками длины 2–4 месяца для сохранения локальной зависимости.
3. **Linear trend + residual bootstrap**: линейный тренд `y_t = a + b t` + бутстрэп остатков.

## Почему ансамбль

Один метод легко промахивается из‑за режима данных. Комбинация прогнозов (forecast combination / model averaging) обычно даёт более устойчивый baseline:

- снижает чувствительность к ошибкам конкретной гипотезы;
- стабилизирует медиану и интервалы;
- делает риск‑метрики менее «хрупкими».

Агрегация делается как **mixture distribution**: симуляции всех методов объединяются в одно распределение, из которого считаются итоговые p10/p50/p90, `probNetNegative` и `uncertainty`.

## Disagreement score

Добавлена метрика **model disagreement** (`0..1`):

- берём точечные p50 каждого метода;
- считаем нормированное стандартное отклонение;
- ограничиваем в диапазоне `0..1`.

Интерпретация в UI:

- **высокое согласие** — score низкий;
- **среднее согласие** — score средний;
- **низкое согласие** — score высокий.

Если disagreement высокий, в Cosmos у планеты History усиливается **halo turbulence**: это означает, что модели «спорят» и уверенность в прогнозе ниже.
